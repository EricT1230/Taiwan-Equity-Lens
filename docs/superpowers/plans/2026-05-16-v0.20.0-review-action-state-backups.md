# v0.20.0 Review Action State Backups Implementation Plan

> **For agentic workers:** Implement task-by-task and update checkboxes as work completes. Keep this release limited to write-before-backup behavior; do not add restore or retention features.

**Goal:** Create a timestamped backup before CLI commands rewrite an existing valid `review_action_state.json`.

**Architecture:** Add a backup helper in `review_action_state.py` that copies original bytes to a unique timestamped backup file. Integrate it into `research action set` and `research action prune-stale --write`, and surface backup paths in CLI output.

**Tech Stack:** Python standard library, `unittest`, existing release doctor.
