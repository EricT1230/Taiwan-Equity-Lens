# v0.9.0 Universe Review Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add deterministic universe-level review metadata and dashboard rendering so users can prioritize research work across the tracked stock universe.

**Architecture:** Extend `research.py` with `build_universe_review(items)` and include it in `research_summary.json`. Extend `dashboard.py` with a new section that renders the review counts, category mix, and attention queue while preserving legacy summary compatibility.

**Tech Stack:** Python standard library only, unittest, static dashboard HTML, existing JSON summary files.

## Task 1: Research Universe Review Model

**Files:**
- Modify: `src/taiwan_stock_analysis/research.py`
- Modify: `tests/test_research.py`

- [ ] **Step 1: Add research tests**

Add tests asserting:

- `build_research_summary(...)[\"universe_review\"]` exists
- category counts include `Uncategorized` for blank category
- counts include total, needs attention, high priority attention, blocked, new, active review
- review buckets include the expected stock ids

- [ ] **Step 2: Add sorting test**

Create a research/workflow fixture where:

- one item is `error` + low priority
- one item is `warning` + high priority
- one item is high priority but ok

Assert the queue order follows status severity first, then priority, then stock id.

- [ ] **Step 3: Implement `build_universe_review(items)`**

Rules:

- normalize blank category to `Uncategorized`
- build `category_counts`, `state_counts`, `priority_counts`
- build `review_buckets`
- build `attention_queue`
- sort with known severity maps

- [ ] **Step 4: Include review object**

Add `\"universe_review\": build_universe_review(items)` to `build_research_summary`.

- [ ] **Step 5: Run focused tests and commit**
