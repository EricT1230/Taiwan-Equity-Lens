# v0.16.0 Review Action Completion Persistence Implementation Plan

> **For agentic workers:** Implement task-by-task and update checkboxes as work completes. Keep generated research summaries deterministic; persist manual review state only in `review_action_state.json`.

**Goal:** Persist review-action completion decisions across regenerated research outputs using a local sidecar state file.

**Architecture:** Add a focused `review_action_state.py` module for loading, writing, and applying persisted state. Extend CLI with `research action list` and `research action set`. Extend dashboard discovery/rendering to overlay sibling state files while preserving static, dependency-free HTML.

**Tech Stack:** Python standard library, `unittest`, existing static dashboard HTML/CSS/JavaScript, existing release doctor.
