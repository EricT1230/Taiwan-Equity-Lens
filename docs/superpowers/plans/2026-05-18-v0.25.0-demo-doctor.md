# v0.25.0 Demo Doctor Implementation Plan

> **For agentic workers:** Build the smallest useful first-run quality gate for `demo quickstart`. Do not add live data checks, browser automation, or auto-repair behavior.

**Goal:** Add `doctor demo` so users can verify that a local quickstart demo produced the expected files and first review-action surfaces.

**Architecture:** Add demo-readiness validation to `taiwan_stock_analysis.doctor`, wire it into `cli.py` under `doctor demo`, document the command, and release as `v0.25.0`.

**Tech Stack:** Python stdlib, existing argparse CLI, existing unittest suite, Markdown docs.
