# v0.45.0 Served HTTP Evidence Smoke Design

v0.45.0 verifies the sector evidence-board interaction through the real local HTTP dashboard server instead of only through helper functions.

## Goal

When the operator uses `dashboard --serve`, the evidence-board action path should work through the same HTTP route used by the browser:

- GET the dashboard,
- read the rendered button payload,
- POST the update,
- fetch the dashboard again,
- confirm the evidence row is deliverable.

## Scope

- Add a non-blocking dashboard server factory for tests.
- Use an ephemeral local port.
- Keep the existing CLI `serve_dashboard()` behavior unchanged.
- Verify the state file write and the post-write dashboard render.

## Non-Goals

- No browser automation dependency.
- No new dashboard state model.
- No investment advice, sector ranking, target price, or allocation guidance.

## Verification

```powershell
python -m unittest tests.test_dashboard_server -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.45.0
```
