# v0.48.0 Evidence Preview Quality Gate Design

v0.48.0 tightens the handoff flow created in v0.47.0. The Evidence Composer can write a file and clear the structural Handoff Quality Gate, but operators still need to see whether the generated evidence is meaningful enough for reviewer confidence.

## Expert Review Lenses

- Product / UX: after pressing the evidence button, the operator should see the output file and the next quality step without reading raw JSON.
- Finance research workflow: evidence must stay framed as handoff support and quality control, not as investment advice.
- Engineering: quality checks should be deterministic, local, and separate from the structural Handoff Quality Gate.

## Scope

- Add a deterministic `evidence_quality` module that checks:
  - local evidence file existence when a path is supplied,
  - named reviewer instead of a default placeholder,
  - specific review note,
  - specific evidence summary,
  - non-investment-advice notice in the evidence content.
- Extend `POST /api/evidence/compose-and-set` to return:
  - quality status,
  - check list,
  - next step,
  - generated evidence preview.
- Update the dashboard Evidence Composer result area to render:
  - Reviewer Confidence,
  - quality check list,
  - output path,
  - markdown excerpt.
- Keep structural gate readiness separate from reviewer confidence. A handoff can be structurally ready while the evidence is still marked `needs_review`.

## Non-Goals

- No LLM scoring or model-generated investment opinion.
- No market-data ranking, buy/sell/hold guidance, target prices, or allocation recommendations.
- No new state store; `review_action_state.json` remains the state source.
- No remote evidence upload.

## Stable Hooks

- `data-evidence-quality-status`
- `data-evidence-preview="true"`
- `data-evidence-preview-content="true"`
- `data-evidence-quality-checks="true"`
