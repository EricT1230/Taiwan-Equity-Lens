# v0.9.0 Universe Review Layer Design

## Goal

v0.9.0 adds a universe-level review layer that turns a research CSV plus workflow status into a compact review console for deciding what to inspect next.

## Product Positioning

Taiwan Equity Lens should remain a research workflow tool, not a portfolio manager and not an investment recommendation engine. The universe review layer should help prioritize research work across many tracked stocks without producing buy, sell, hold, allocation, or ranking recommendations.

The layer should answer:

1. Which names need attention first?
2. Which categories or states dominate the research universe?
3. Which high-priority items have workflow or data-reliability issues?
4. Which items are blocked, new, or ready for manual review?
5. What should the user open next in the dashboard?

## Scope

Included:

- `universe_review` object in `research_summary.json`
- deterministic attention queue sorted by workflow urgency and research priority
- category / state / priority breakdowns
- review buckets for blocked, new, active review, and high-priority attention items
- dashboard section for universe-level review
- documentation, release notes, version bump, and tests

Excluded:

- portfolio allocation
- recommendation score
- buy / sell / hold output
- external market/news/technical data
- database or hosted dashboard
- changes to metric or valuation calculations

## Universe Review Data Model

Add this shape to `research_summary.json`:

```json
{
  "universe_review": {
    "counts": {
      "total": 3,
      "needs_attention": 2,
      "high_priority_attention": 1,
      "blocked": 1,
      "new": 1,
      "active_review": 1
    },
    "category_counts": {
      "Semiconductor": 2,
      "Uncategorized": 1
    },
    "state_counts": {
      "review": 1,
      "new": 1,
      "blocked": 1
    },
    "priority_counts": {
      "high": 1,
      "medium": 1,
      "low": 1
    },
    "attention_queue": [
      {
        "stock_id": "2330",
        "company_name": "TSMC",
        "category": "Semiconductor",
        "priority": "high",
        "research_state": "review",
        "workflow_status": "warning",
        "reliability_status": "warning",
        "attention_reasons": ["data reliability is warning"],
        "review_bucket": "high_priority_attention"
      }
    ],
    "review_buckets": {
      "high_priority_attention": ["2330"],
      "blocked": ["9999"],
      "new": ["2303"],
      "active_review": ["2330"]
    }
  }
}
```

## Sorting Rules

Attention queue order should be deterministic:

1. workflow status severity: `error`, `warning`, `skipped`, `ok`
2. reliability status severity: `error`, `warning`, `skipped`, `ok`
3. priority: `high`, `medium`, `low`
4. research state urgency: `blocked`, `review`, `new`, `watching`, `done`
5. stock id

Only items with attention reasons, high priority, blocked state, new state, or non-ok workflow/reliability status should appear in the attention queue.

## Review Buckets

Buckets should be deterministic and non-recommendational:

- `high_priority_attention`: priority is high and item has attention reasons or non-ok status
- `blocked`: research state is blocked
- `new`: research state is new
- `active_review`: research state is review

One stock can appear in multiple buckets.

## Dashboard Behavior

When `research_summary.json` contains `universe_review`, the dashboard should render a new `Universe Review` section near the research summary.

The section should show:

- total tracked items
- needs attention
- high-priority attention
- blocked
- new
- active review
- category counts
- top attention queue table

Older summaries without `universe_review` should still render normally.

## Architecture

Keep the logic local to the research and dashboard layers:

- `src/taiwan_stock_analysis/research.py`
  - add `build_universe_review(items)`
  - include `universe_review` in `build_research_summary`
- `src/taiwan_stock_analysis/dashboard.py`
  - add a universe review section renderer
  - keep existing research summary rendering intact

No new third-party dependencies.

## Error Handling

- Missing category should be counted as `Uncategorized`.
- Invalid / missing `attention_reasons` should be treated as an empty list.
- Unknown priority or status values should sort after known values.
- Dashboard should tolerate missing or malformed `universe_review`.

## Testing

Required coverage:

- research summary includes universe review counts, category counts, queues, and buckets
- attention queue sort order is deterministic
- empty / missing category becomes `Uncategorized`
- dashboard renders universe review section when present
- dashboard remains compatible with legacy research summaries without `universe_review`
- existing tests continue passing
