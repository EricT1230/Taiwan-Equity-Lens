# v0.20.0 Review Action State Backups Design

## Goal

Add automatic backup creation before CLI commands rewrite `review_action_state.json`. v0.19.0 introduced explicit stale-state pruning; v0.20.0 adds a safety layer so state-changing commands leave the previous file available for manual recovery.

## User Interface

No new user command is added. Existing write commands gain backup behavior:

```powershell
python -m taiwan_stock_analysis.cli research action set research-dist/review_action_state.json 2330 source-audit-manual-review --status done
python -m taiwan_stock_analysis.cli research action prune-stale research-dist/research_summary.json --state research-dist/review_action_state.json --write
```

When a backup is created, the CLI prints:

```text
Backup review action state: research-dist/review_action_state.json.bak-20260516T173000Z
```

## Backup Naming

Backups use a UTC timestamp suffix:

```text
review_action_state.json.bak-YYYYMMDDTHHMMSSZ
```

If multiple backups are created in the same second, the implementation may append a numeric suffix such as `.1` to avoid overwriting an existing backup.

## Data Flow

1. A write-capable command loads and validates the state file using the existing loader.
2. If the state file exists and loaded successfully, copy the original file bytes to a timestamped backup path.
3. Write the new normalized state file.
4. Print the backup path before or alongside the existing write/prune confirmation.

## Safety And Error Handling

- If the state file does not exist, no backup is created.
- If the state file is invalid JSON, the command does not write and does not create a backup.
- Backup content must be the pre-write file content, not the normalized or post-write payload.
- Backup creation failure should fail the write command before modifying the state file.
- Existing state schema and command behavior stay unchanged aside from the extra backup line.

## Scope

In scope:

- Backup helper in `review_action_state.py`.
- Backup integration for `research action set`.
- Backup integration for `research action prune-stale --write`.
- Unit tests for backup naming and content preservation.
- CLI tests for backup output and no-backup cases.
- Release metadata and docs for v0.20.0.

Out of scope:

- `restore-backup` or undo commands.
- Dashboard backup controls.
- Automatic backup cleanup or retention policy.
- Backups for unrelated workflow outputs.

## Testing

Unit coverage should verify:

- Backup path uses the expected suffix format.
- Backup helper preserves original file bytes.
- Existing backup filenames are not overwritten.

CLI coverage should verify:

- `research action set` creates a backup when updating an existing state file.
- `research action prune-stale --write` creates a backup when pruning an existing state file.
- Missing state file writes do not create a backup.
- Invalid state JSON does not create a backup.
- Backup content is the original pre-write state.

Release verification should include focused state/CLI tests, the full test suite, Python 3.10 smoke, release doctor, and a demo workflow that shows backup creation before both `set` and `prune-stale --write`.
