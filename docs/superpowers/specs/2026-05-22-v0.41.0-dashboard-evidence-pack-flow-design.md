# v0.41.0 Dashboard Evidence Pack Flow Design

v0.41.0 turns the Handoff Evidence Pack into a first-screen dashboard workflow. v0.40.0 created the pack from CLI and doctor commands; this release lets the operator create or copy that workflow from the Expert Agent Console.

## Goal

When the dashboard is open, the operator should be able to answer:

- Can I generate the handoff evidence pack now?
- Which file paths will be produced?
- Which stock/action still needs evidence?
- What exact local evidence file should I create?
- What button should I press next?

The output remains a research workflow artifact only and does not provide investment advice.

## Scope

- Keep `review_action_state.json` as the only state store.
- Add a served dashboard API endpoint for pack generation.
- Add a dashboard workflow panel to the Expert Agent Console.
- Keep static dashboards usable by copying the equivalent CLI command.
- Show generated Markdown, HTML, and JSON summary paths after the API writes the pack.
- Show missing or invalid evidence guidance derived from the same Handoff Quality Gate blockers.
- Add a dashboard industry rotation map preview that groups research items by `category` and shows research-delivery pressure, not investment opportunity.

## API Contract

`POST /api/handoff-pack/write`

Input:

```json
{
  "research_summary_path": "research-dist/research_summary.json",
  "state_path": "research-dist/review_action_state.json",
  "output_dir": "research-dist/handoff-pack",
  "format": "both",
  "blocker_limit": 10
}
```

Output:

```json
{
  "ok": true,
  "ready": false,
  "gate_status": "blocked",
  "blocker_count": 3,
  "evidence_missing_count": 1,
  "invalid_evidence_count": 0,
  "markdown_path": "research-dist/handoff-pack/handoff-pack.md",
  "html_path": "research-dist/handoff-pack/handoff-pack.html",
  "summary_path": "research-dist/handoff-pack/handoff_pack_summary.json"
}
```

Path handling follows the existing served-dashboard safety rule: requested paths must resolve inside one of the served dashboard roots.

## Dashboard Behavior

- Served mode:
  - Button text: `產出 Evidence Pack`
  - Calls `/api/handoff-pack/write`
  - Displays ready/blocked status and generated output paths.

- Static mode:
  - Button text: `複製 Evidence Pack 指令`
  - Copies `python -m taiwan_stock_analysis.cli research handoff-pack ...`
  - Tells the operator to run it in PowerShell and refresh the dashboard.

- Evidence guidance:
  - Missing evidence blockers show missing fields plus a suggested path like `research-dist/evidence/2330-reliability-warning.md`.
  - Invalid evidence blockers show the current invalid reference plus a suggested local file path.

- Industry rotation map:
  - Groups by the research `category` field from `research_summary.json`.
  - Uses blocker count, missing evidence, invalid evidence, stale state, open review actions, high priority count, and attention count to order cards.
  - `前往這個產業阻塞` filters the existing Review Actions table to the target stock/action.
  - Copy must state that the map is for handoff triage only, not industry strength, price momentum, or buy/sell/hold guidance.
