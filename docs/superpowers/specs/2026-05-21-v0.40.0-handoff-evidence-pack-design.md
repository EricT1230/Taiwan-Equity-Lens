# v0.40.0 Handoff Evidence Pack Design

## Goal

Turn the v0.39.0 evidence-required handoff gate into a concrete review artifact. After a dashboard or CLI reviewer handles blockers, the user should be able to generate one local pack that answers:

- Can this research be handed off?
- Which blockers remain?
- Which handled blockers have evidence?
- Who reviewed them?
- Where is the supporting evidence?
- Is any local evidence reference missing?
- Is this still clearly not investment advice?

## Scope

- Add a deterministic handoff pack generator.
- Keep the existing research pack unchanged.
- Keep review-action state schema compatible.
- Strengthen `doctor handoff` only when local evidence references can be checked.
- Surface generated handoff pack outputs in the dashboard.

## Data Inputs

- `research_summary.json`
- optional `review_action_state.json`
- local evidence files referenced by `evidence_url`

## Outputs

- `handoff-pack.md`
- `handoff-pack.html`
- `handoff_pack_summary.json`

The JSON summary records gate status, blocker counts, evidence gap counts, invalid evidence reference counts, output paths, and traceability metadata.

## Evidence Validation

- Empty `note`, `reviewer`, `evidence_url`, or `updated_at` remains an evidence-required gap.
- `http` and `https` evidence references are accepted as remote references.
- Local paths are resolved relative to the directory containing `research_summary.json`.
- Missing local files are reported as `invalid_evidence`.

## Dashboard

The dashboard discovers `handoff-pack/` outputs and shows their status, ready flag, blocker count, missing evidence count, invalid evidence count, and links to Markdown, HTML, and JSON summary outputs.

## Non-Advice Boundary

The pack must use research workflow language only. It must not present buy, sell, hold, target-price, or portfolio allocation guidance.

## Verification

- Unit coverage for handoff gate invalid evidence references.
- Unit coverage for handoff pack context, Markdown/HTML/summary generation.
- CLI coverage for `research handoff-pack` and `doctor handoff --write-pack`.
- Dashboard coverage for handoff pack output rendering.
- Release doctor and full test discovery before tagging.
