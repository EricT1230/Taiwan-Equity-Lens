# v0.7.0 Memo Intelligence Upgrade Design

## Goal

v0.7.0 upgrades single-stock research memos from structured factual summaries into deterministic review-ready research drafts that combine concise synthesis with explicit follow-up work.

## Product Positioning

Taiwan Equity Lens should keep a clear split:

- Research packs summarize the whole research run.
- Research memos summarize one stock in a format that is easier to review, discuss, and hand off.

The memo must remain deterministic and factual. It should not provide buy, sell, hold, ranking, or allocation recommendations.

## Scope

Included:

- Executive summary section
- Key observations section
- Catalysts section
- Risks section
- Open questions section
- Next research actions section with grouped actions
- Updated Markdown and HTML memo rendering
- Documentation, release notes, version bump, and tests

Excluded:

- LLM prose generation
- external news or alternative data
- PDF / DOCX export
- recommendation labels or final thesis verdicts
- changes to metric, scorecard, or valuation formulas

## Memo Structure

The memo should use this section order:

1. Title
2. Executive Summary
3. Research Metadata
4. Key Observations
5. Catalysts
6. Risks
7. Open Questions
8. Latest Metrics Snapshot
9. Valuation Context
10. Quality Scorecard
11. Diagnostics
12. Next Research Actions
13. Source Links
14. Disclaimer

## Section Design

### Executive Summary

The executive summary should contain 3 to 5 concise bullet points chosen deterministically from available data.

Candidate sources:

- latest metrics snapshot
- scorecard total score / confidence
- valuation presence or missing assumptions
- reliability status
- research state / attention reasons

Example output shape:

- Revenue and profitability metrics are available for the latest year.
- Quality scorecard is `78` with `90` confidence.
- Valuation scenarios are present, but assumptions should still be reviewed.
- Workflow reliability is `warning`; verify flagged data issues before handoff.

### Key Observations

Group observations under:

- Operating
- Profitability
- Financial Health

The implementation should reuse existing deterministic insight data when available. If insight groups are missing, fallback observations may be derived conservatively from metrics or render as `-`.

### Catalysts

Catalysts are not investment catalysts in the recommendation sense. They are factual positive research signals worth tracking.

Examples:

- improving or available revenue profile
- positive margin / ROE information
- positive free-cash-flow margin
- scorecard confidence present and not low

Language must remain observational:

- "Track whether revenue momentum persists."
- "Positive free-cash-flow margin is available for review."

### Risks

Risks should be derived from:

- diagnostics issues
- reliability warnings or errors
- missing valuation assumptions
- negative or missing core metrics

Examples:

- "Price-source warning remains unresolved."
- "Dividend or book-value assumptions are missing."
- "Diagnostics reported missing dividend data."

### Open Questions

Questions should identify unresolved review needs:

- why a warning exists
- whether assumptions need manual entry
- whether filings should be checked
- whether research state should advance

Questions must be phrased as actual questions and should never answer themselves.

### Next Research Actions

Replace the current flat checklist with grouped deterministic actions:

- Data Checks
- Valuation Checks
- Thesis Checks

Examples:

- Data Checks
  - Review workflow reliability warning.
  - Inspect diagnostics issues and retry hints.
- Valuation Checks
  - Confirm valuation assumptions are complete.
  - Review target-price scenarios as scenarios only.
- Thesis Checks
  - Confirm research notes and state are still accurate.
  - Compare the memo with source filings before handoff.

## Data Requirements

No public JSON schema change is required for existing workflow outputs.

The memo renderer should work from the current context:

- `analysis`
- `research_item`
- optional workflow / research file paths

Derived section helpers should use only this current context.

## Architecture

Continue using `src/taiwan_stock_analysis/memo.py`, but split additional logic into focused helpers inside the module:

- `_executive_summary_lines(ctx)`
- `_observation_groups(ctx)`
- `_catalyst_lines(ctx)`
- `_risk_lines(ctx)`
- `_open_question_lines(ctx)`
- `_next_action_groups(ctx)`

Markdown and HTML should render from the same helper outputs wherever practical, so content remains consistent across formats.

## Output Rules

- Missing sections should render a concise empty state such as `-`.
- Markdown table escaping behavior must remain intact.
- HTML escaping must remain intact.
- New text should be deterministic and stable across runs with identical inputs.
- New text must not use recommendation language.
- Existing disclaimer should remain unchanged unless expanded without weakening guardrails.

## Error Handling

- Existing invalid analysis JSON behavior remains unchanged.
- Absence of insights, scorecard, valuation, or diagnostics should not break rendering.
- The renderer should prefer a partial memo with visible empty states over failing.

## Testing

Required coverage:

- Markdown memo includes all new sections in the correct order
- HTML memo includes all new sections
- executive summary reflects scorecard / valuation / reliability context
- catalysts and risks appear when supported by data
- open questions appear when warnings or missing assumptions exist
- next research actions render grouped actions
- empty or missing optional context produces `-` rather than failures
- existing escaping tests continue passing
- research memo batch generation still works

## Release Criteria

v0.7.0 is ready when:

1. Single-stock memos read like deterministic review-ready research drafts.
2. New sections render in Markdown and HTML.
3. Risks, questions, and actions respond to existing diagnostics, reliability, and valuation context.
4. The memo remains recommendation-free and clearly labeled as research support only.
5. Existing research memo batch flow continues to work.
6. Tests cover the new sections and no existing suite regressions remain.
