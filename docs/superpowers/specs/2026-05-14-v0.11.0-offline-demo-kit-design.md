# v0.11.0 Offline Demo Kit Design

## Summary

v0.11.0 will add a deterministic offline demo kit so a new user can clone the repository, install the package, run one command, and inspect a complete workflow dashboard without network access. The current README demo still fetches live Goodinfo financial pages unless the user provides their own fixtures. That makes the first-run experience dependent on external site availability and request behavior.

The release will ship small synthetic fixture HTML files under `examples/fixtures/`. These fixtures will not copy live Goodinfo content. They will contain minimal table structures that match the parser contract and produce realistic but artificial metrics for example stocks.

## Goals

- Provide a reliable offline workflow demo using committed example fixtures.
- Update README and examples documentation so the primary demo command is fully local.
- Add a CI-covered smoke test for the demo command.
- Keep fixture content synthetic and small.
- Preserve existing live-fetch commands for real research use.

## Non-Goals

- Do not ship real Goodinfo pages or copyrighted source pages.
- Do not add new analysis metrics.
- Do not change scorecard, valuation, memo, pack, or dashboard semantics.
- Do not introduce a database, cache service, or external dependency.
- Do not change the default behavior of normal live commands.

## User-Facing Behavior

The primary demo command becomes:

```powershell
python -m taiwan_stock_analysis.cli research run examples/research.csv --fixture-root examples/fixtures --output-dir demo-dist --offline-prices
```

The command should produce:

- `demo-dist/research_watchlist.csv`
- `demo-dist/reports/batch_summary.json`
- `demo-dist/valuation.csv`
- `demo-dist/valuation-reports/batch_summary.json`
- `demo-dist/comparison/comparison.json`
- `demo-dist/comparison/comparison.html`
- `demo-dist/research_summary.json`
- `demo-dist/memos/memo_summary.json`
- `demo-dist/memos/*_memo.md`
- `demo-dist/memos/*_memo.html`
- `demo-dist/packs/pack_summary.json`
- `demo-dist/packs/research-pack.md`
- `demo-dist/packs/research-pack.html`
- `demo-dist/workflow_summary.json`
- `demo-dist/dashboard.html`

The user opens:

```text
demo-dist/dashboard.html
```

## Fixture Layout

Create:

```text
examples/fixtures/
|-- 2330/
|   |-- IS_YEAR.html
|   |-- BS_YEAR.html
|   `-- CF_YEAR.html
|-- 2303/
|   |-- IS_YEAR.html
|   |-- BS_YEAR.html
|   `-- CF_YEAR.html
`-- 2454/
    |-- IS_YEAR.html
    |-- BS_YEAR.html
    `-- CF_YEAR.html
```

The fixtures should be intentionally synthetic:

- Add an HTML comment near the top: `Synthetic fixture for Taiwan Equity Lens tests and demos.`
- Use only minimal tables required by the parser.
- Include two years of data.
- Include enough fields for core profitability, leverage, cash-flow, EPS, and scorecard calculations.

Update:

- README example file list to mention `examples/fixtures`.
- `examples/README.md` demo commands.

## Tests

Add a focused smoke test that runs the public offline demo command against committed examples:

```powershell
python -m taiwan_stock_analysis.cli research run examples/research.csv --fixture-root examples/fixtures --output-dir .tmp-cli-test/offline-demo-dist --offline-prices
```

The test should assert:

- exit code is `0`
- `dashboard.html` exists
- `research_summary.json` exists
- `workflow_summary.json` exists
- `memos/memo_summary.json` exists
- `packs/pack_summary.json` exists
- comparison output exists, because the example research universe has at least two fixture-backed stocks

## Error Handling

The demo should use existing workflow error handling. Missing fixture files should surface as per-stock failures in summaries, not crash the entire research workflow. The committed fixtures should cover all stocks in `examples/research.csv` so the demo path is green by default.

## Acceptance Criteria

- The offline demo command completes without network access.
- The generated dashboard, research summary, memo summary, pack summary, and comparison output exist.
- README primary demo no longer depends on live Goodinfo fetches.
- `python -m taiwan_stock_analysis.cli doctor release --version 0.11.0` passes after release metadata updates.
- `python -m unittest discover -s tests -v` passes.
- `uv run --python 3.10 python -m unittest discover -s tests -v` passes.
