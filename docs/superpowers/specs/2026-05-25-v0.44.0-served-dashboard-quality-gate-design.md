# v0.44.0 Served Dashboard Quality Gate Design

v0.44.0 verifies the v0.43.0 sector evidence-board interaction path. The goal is to prove the dashboard-rendered button payload can reach the same state update path used by served dashboards.

## Goal

When a served dashboard shows a sector evidence row, the operator should be able to trust that pressing `補證並標記完成` maps to a valid review-action state update.

## Scope

- Reuse the existing review-action state API helper.
- Build the test from rendered dashboard HTML rather than hand-written button payloads.
- Confirm the state file records status, note, reviewer, and evidence URL.
- Re-render the dashboard after the write and verify the stock evidence row becomes `證據可交付`.

## Non-Goals

- No browser automation dependency.
- No new state store.
- No investment advice, sector ranking, price target, or allocation guidance.

## Verification

```powershell
python -m unittest tests.test_dashboard_server -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.44.0
```
