# v0.6.0 Research Pack Design

## Goal

v0.6.0 adds a deterministic research pack that consolidates research workflow outputs into a single local review artifact.

## Product Positioning

Taiwan Equity Lens should remain a local, file-based research workflow tool. The research pack should make an existing run easier to review, archive, and hand off without adding hosted services, databases, or recommendation logic.

The pack should help a user answer:

1. What happened in this research run?
2. Which stocks need attention first?
3. Which generated outputs should be opened next?
4. What reliability or workflow issues affect interpretation?
5. Where are the per-stock memo, dashboard, workflow, and summary files?

## Scope

Included:

- `research pack` command for generating a consolidated pack from `research_summary.json`
- Markdown pack output
- Optional HTML pack output using the same content model
- `research run` integration that writes packs by default unless disabled
- Pack discovery and links in the static dashboard
- Pack summary JSON for generated and skipped outputs
- Documentation, release notes, version bump, and tests

Excluded:

- PDF / DOCX export
- LLM prose generation
- Investment recommendation labels or ranking recommendations
- New market data sources
- Database or hosted web application
- Changes to existing valuation or scoring formulas

## User Experience

### Standalone Pack Generation

Generate a pack from an existing research workflow directory:

```powershell
python -m taiwan_stock_analysis.cli research pack research.csv --workflow-dir research-dist --output-dir research-dist/packs
```

The command should read:

- `research-dist/research_summary.json`
- `research-dist/workflow_summary.json` when present
- `research-dist/memos/memo_summary.json` when present

and write a pack that links back to those local files plus per-stock memo outputs when they exist.

### Research Run Integration

The existing command:

```powershell
python -m taiwan_stock_analysis.cli research run research.csv --output-dir research-dist --offline-prices
```

should generate:

- reports
- research summary
- memo outputs
- research pack outputs

unless the user passes:

```powershell
--skip-packs
```

Pack generation should run after research summary generation and after memo generation so the pack can reference memo artifacts when available.

## Pack Content

Each pack should contain deterministic sections:

1. Title
   - research pack title and source research CSV path
2. Run Overview
   - total item count
   - research-state counts
   - priority counts
   - attention count
3. Reliability Overview
   - workflow-level reliability counts when available
   - pack-level summary of warning / error presence
4. Priority Review Queue
   - attention items first
   - then high-priority rows
   - stock id, company name, state, priority, workflow status, reliability status
5. Generated Outputs
   - dashboard path
   - workflow summary path
   - research summary path
   - memo summary path when present
6. Per-Stock Research Index
   - one row per research item
   - memo Markdown / HTML links when present
   - per-stock attention reasons
7. Review Checklist
   - deterministic checks based on missing pack inputs or warning counts
8. Disclaimer
   - explicit statement that the pack is research workflow support only and not investment advice

## Data Model

Use plain dictionaries so the pack renderer can be tested without additional dependencies.

Recommended context shape:

```json
{
  "research_csv_path": "research.csv",
  "research_summary_path": "research-dist/research_summary.json",
  "workflow_summary_path": "research-dist/workflow_summary.json",
  "memo_summary_path": "research-dist/memos/memo_summary.json",
  "dashboard_path": "research-dist/dashboard.html",
  "counts": {
    "total": 3,
    "needs_attention": 2
  },
  "research_state_counts": {
    "review": 2,
    "monitor": 1
  },
  "priority_counts": {
    "high": 1,
    "medium": 2
  },
  "workflow_reliability": {
    "ok": 1,
    "warning": 1,
    "error": 0,
    "skipped": 1
  },
  "items": [
    {
      "stock_id": "2330",
      "company_name": "TSMC",
      "research_state": "review",
      "priority": "high",
      "workflow_status": "ok",
      "reliability_status": "warning",
      "attention_reasons": ["data reliability is warning"],
      "memo_markdown_path": "research-dist/memos/2330_memo.md",
      "memo_html_path": "research-dist/memos/2330_memo.html"
    }
  ]
}
```

## Architecture

Add a focused pack module:

- `src/taiwan_stock_analysis/pack.py`
  - loads research summary JSON
  - optionally reads workflow and memo summaries
  - builds a pack context
  - renders Markdown
  - renders HTML
  - writes single pack output and `pack_summary.json`

Wire it through:

- `src/taiwan_stock_analysis/cli.py`
  - adds `research pack`
  - adds `--skip-packs` to `research run`
- `src/taiwan_stock_analysis/dashboard.py`
  - discovers pack outputs
  - renders pack links in the dashboard

No new third-party dependencies should be introduced.

## Output Rules

- Markdown should be readable in GitHub and local editors.
- HTML should be static and restrained, following existing report/dashboard tone.
- Missing optional outputs should render as `-`.
- Packs should never imply buy / sell / hold recommendations.
- The review queue should sort attention items first, then priority order `high`, `medium`, `low`, then stock id.
- Pack output paths should be deterministic:
  - `research-pack.md`
  - `research-pack.html`
  - `pack_summary.json`

## Error Handling

- Missing research summary should raise a clear `FileNotFoundError`.
- Invalid research summary JSON should raise a clear `ValueError`.
- Missing workflow summary or memo summary should not fail pack generation.
- Batch-like writing should still produce a pack even if some per-stock memo links are unavailable.
- Pack generation inside `research run` should not fail the overall research run; failures should be recorded in `pack_summary.json`.

## Pack Summary

Pack generation should write:

```json
{
  "output_dir": "research-dist/packs",
  "markdown_path": "research-dist/packs/research-pack.md",
  "html_path": "research-dist/packs/research-pack.html",
  "status": "ok",
  "warnings": []
}
```

When optional pack inputs are missing, status can remain `ok` with warning messages in `warnings`.

## Testing

Required coverage:

- render Markdown pack with overview, priority review queue, output links, per-stock index, checklist, and disclaimer
- render HTML pack with escaped user-provided values
- missing optional memo/workflow summaries render graceful empty values
- invalid research summary raises `ValueError`
- `research pack` writes Markdown, HTML, and `pack_summary.json`
- `research run` writes packs by default
- `research run --skip-packs` skips pack generation
- dashboard discovers and links pack outputs
- existing tests continue passing

## Release Criteria

v0.6.0 is ready when:

1. Users can generate a consolidated research pack from an existing research workflow directory.
2. `research run` produces pack outputs unless disabled.
3. Dashboard links to pack outputs.
4. Pack text stays factual, deterministic, and free of recommendation language.
5. Missing optional workflow or memo inputs do not block pack generation.
6. Tests cover rendering, CLI integration, dashboard discovery, and error handling.
7. Documentation explains the pack workflow clearly.
