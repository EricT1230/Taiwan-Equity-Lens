# v0.22.0 Review Action State Backup List Design

## Goal

Add a read-only CLI command that lists backup files available for a review-action state file. v0.20.0 creates backups and v0.21.0 restores an explicit backup path; v0.22.0 makes the restore workflow easier by showing the backup candidates without requiring manual directory inspection.

## Command

Add a new subcommand:

```bash
python -m taiwan_stock_analysis.cli research action backups STATE_PATH [--json]
```

- `STATE_PATH` is the target state file name, usually `review_action_state.json`.
- The command searches `STATE_PATH.parent` for files named `{STATE_PATH.name}.bak-*`.
- The command is read-only and does not validate or mutate the current state file.

## Default Text Output

Default output is a tab-separated table sorted newest first:

```text
created_at	size	path
2026-05-16T18:00:00Z	280	research-dist/review_action_state.json.bak-20260516T180000Z
unknown	272	research-dist/review_action_state.json.bak-source
```

Fields:

- `created_at`: parsed from the backup filename suffix when it matches `YYYYMMDDTHHMMSSZ`; otherwise `unknown`.
- `size`: file size in bytes.
- `path`: backup path as a string.

When no backups are found, print only the header row and exit `0`.

## JSON Output

With `--json`, print a deterministic JSON object:

```json
{
  "state_path": "research-dist/review_action_state.json",
  "backups": [
    {
      "created_at": "2026-05-16T18:00:00Z",
      "size": 280,
      "path": "research-dist/review_action_state.json.bak-20260516T180000Z"
    }
  ]
}
```

The JSON output uses `ensure_ascii=False`, `indent=2`, and sorted object keys where existing project style calls for deterministic output.

## Sorting

Sort newest first.

1. Backups with a parsed timestamp sort before unknown-timestamp backups.
2. Parsed timestamps sort descending.
3. Same timestamp and unknown-timestamp groups sort by path string descending for deterministic output.

This keeps the most likely restore candidate at the top while making filename collision suffixes deterministic.

## Code Shape

Add a helper in `review_action_state.py`:

```python
list_review_action_state_backups(path: Path) -> list[dict[str, object]]
```

Each row should include:

- `created_at`: string
- `created_at_sort`: internal `datetime | None` or equivalent sorting value
- `size`: integer
- `path`: string or `Path`

The public CLI output should not expose internal sort-only fields. If needed, use a private helper for parsing backup timestamps and keep the returned dictionaries ready for display.

## Error Handling

- Missing `STATE_PATH` is allowed because backups can exist even if the target state file has been deleted.
- Missing parent directory produces an empty list.
- File stat failures should be skipped rather than failing the whole list command.
- The command does not validate backup JSON. It is an inventory command, not a restore dry-run.

## Non-Goals

- No automatic latest-backup restore.
- No backup deletion or cleanup policy.
- No dashboard backup list.
- No backup JSON validation in the list command.

## Acceptance Criteria

- `research action backups STATE_PATH` lists matching backups newest first.
- `research action backups STATE_PATH --json` emits deterministic JSON.
- Missing target state files and missing directories are handled without error.
- Unrelated files are ignored.
- Focused state/CLI tests, full suite, Python 3.10 smoke, release doctor, and a backup-list demo all pass.
