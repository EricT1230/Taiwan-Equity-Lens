# v0.46.0 Guided Next Action Loop Design

v0.46.0 turns the Expert Agent Console into a single-next-step workflow. v0.45.0 proved the served HTTP update path works; this release makes the dashboard tell the operator exactly which primary button to press, what changed after pressing it, and what remains before handoff.

## Expert Review Lenses

- Product / UX: the first dashboard screen must answer "what do I press next?" without requiring the user to read the Review Actions table first.
- Finance research workflow: all copy remains handoff-quality and evidence-control language, not investment advice.
- Engineering: the served HTTP result must include updated gate counts so the UI can show a concrete post-action result.

## Scope

- Add a Next Action Workbench inside the Expert Agent Console.
- Show one recommended primary action:
  - handle the highest-priority blocker when the gate is blocked,
  - generate the Evidence Pack when the gate is ready.
- Display post-action result text with stock, action, status, remaining open actions, remaining gate blockers, and the next step.
- Keep the industry map and review-action table as drilldown surfaces; the workbench remains the first action loop.
- Extend served-dashboard HTTP coverage to verify the workbench button payload can update state and re-render to the ready next action.

## Non-Goals

- No investment advice, buy/sell/hold language, target prices, sector attractiveness ranking, or allocation guidance.
- No new expert-label taxonomy in this release.
- No live market-data or brokerage integration.

## Stable Hooks

- `data-next-action-workbench="true"`
- `data-next-action-kind`
- `data-next-action-primary="true"`
- `data-next-action-result="true"`
- `data-next-action-remaining="true"`
