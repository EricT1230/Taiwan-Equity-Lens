# v0.37.0 Handoff Quality Gate Design

## Goal

v0.37.0 turns the v0.35/v0.36 Expert Agent Console into a deterministic handoff gate shared by the CLI and dashboard. A user should be able to run one command or open the dashboard and know whether the current research can move into human handoff review.

## Scope

- Add a reusable gate builder that accepts a research summary plus optional `review_action_state`.
- Add `doctor handoff <research_summary.json>` with text and JSON output.
- Use the same gate in the Expert Agent Console.
- Block on:
  - open state-overlaid review actions,
  - stale sidecar state,
  - missing required actions inferred from source, workflow, reliability, valuation, fundamental-review, and research-state signals.
- Preserve the non-investment-advice notice.

## Non-Goals

- Do not create a second review-action state system.
- Do not make the dashboard an investment recommendation surface.
- Do not require network access.
- Do not change existing review-action command contracts.

## Acceptance Tests

- Handoff gate passes when generated actions are all handled and no stale state remains.
- Handoff gate fails on open actions.
- Handoff gate fails on stale sidecar entries.
- Handoff gate fails when a required action is missing from a summary that contains gate-worthy risk signals.
- `doctor handoff` returns nonzero and prints Top 3 blockers when blocked.
- The Expert Agent Console uses gate status, counts, next step, and Top 3 blockers.
- Existing dashboard and review-action tests continue to pass.
