# v0.12.0 Research Quality Upgrade Design

## Goal

v0.12.0 upgrades Taiwan Equity Lens from a workflow report generator into a more complete deterministic research workflow by adding explicit research assumptions, valuation confidence context, and methodology documentation.

## Product Positioning

The tool should help a user structure fundamental research without pretending to make an investment decision. It should make the research trail clearer:

- What is the working thesis?
- What risks or triggers should be watched?
- Which valuation scenarios are complete enough to review?
- What follow-up work remains before handoff?

The output must remain deterministic. It must not add buy, sell, hold, ranking, allocation, or final recommendation labels.

## Scope

Included:

- Optional research CSV fields for `thesis`, `key_risks`, `watch_triggers`, and `follow_up_questions`.
- Backward-compatible parsing for older research CSV files.
- Valuation scenario summary with fair-value range, margin-of-safety context, and valuation confidence.
- Memo sections that surface thesis, risks, triggers, questions, and valuation confidence.
- Research pack sections that summarize thesis coverage and follow-up needs across the universe.
- `docs/research-methodology.md` explaining the deterministic research method.
- README, examples, changelog, release notes, and package version update for `v0.12.0`.

Excluded:

- LLM-generated prose.
- External news, consensus estimates, or alternative data.
- Portfolio allocation, trade execution, or recommendation labels.
- Changes to financial statement parsing.
- Changes to the existing scorecard scoring formula.

## Research CSV Design

The research CSV keeps existing columns and adds optional research-quality fields:

```text
stock_id,company_name,category,priority,research_state,notes,thesis,key_risks,watch_triggers,follow_up_questions
```

Rules:

- Existing CSV files with only the old columns remain valid.
- Missing new fields are normalized to empty strings.
- The template includes concise sample values so a new user sees how to use the fields.
- These fields are carried into `research_summary.json` items and downstream memo / pack outputs.

## Valuation Quality Design

The valuation payload adds a `scenario_summary` object:

```json
{
  "fair_value_range": {
    "low": 600.0,
    "base": 1000.0,
    "high": 1375.0
  },
  "margin_of_safety_percent": 0.0,
  "valuation_confidence": {
    "score": 100,
    "label": "high",
    "reasons": [
      "price_available",
      "base_eps_available",
      "target_pe_range_complete"
    ]
  }
}
```

Confidence is based only on assumption completeness, not whether a stock is attractive:

- +25 price is available
- +25 base EPS scenario is available
- +25 all target PE assumptions are available
- +25 low, base, and high target prices are available

Labels:

- `high`: score >= 75
- `medium`: score >= 50 and < 75
- `low`: score < 50

`margin_of_safety_percent` is `(base_target_price - price) / base_target_price * 100`. If price or base target price is missing or zero, it is `null`.

## Memo Design

Memos add a `Thesis Snapshot` section after `Research Metadata`.

The section displays:

- Thesis
- Key risks
- Watch triggers
- Follow-up questions

The valuation section also adds:

- Fair-value range
- Margin-of-safety context
- Valuation confidence score, label, and reasons

Existing risk and open-question helpers should incorporate the new research fields when present. Missing fields render as `-`.

## Research Pack Design

Research packs add a `Research Quality Overview` section.

The section summarizes:

- How many items have a thesis.
- How many items have watch triggers.
- How many items have follow-up questions.
- Which high-priority items are missing thesis or follow-up fields.

The per-stock research index adds compact columns for thesis and follow-up coverage.

## Documentation

Add `docs/research-methodology.md` with:

- Data source boundaries.
- Metric and scorecard interpretation.
- Valuation scenario interpretation.
- Research CSV field meanings.
- Suggested review workflow.
- Explicit non-advice disclaimer.

Update README and examples to link the methodology page and show the richer research CSV workflow.

## Error Handling

- Old research CSV files remain accepted.
- Missing optional research-quality fields do not fail workflow execution.
- Invalid required fields keep the current validation behavior.
- Missing valuation inputs produce low confidence and visible empty states, not crashes.

## Testing

Required coverage:

- Old research CSV files still parse.
- New research fields parse, normalize, and round-trip into `research_summary.json`.
- Research template includes all new fields.
- Valuation summary computes fair-value range, margin-of-safety context, and confidence.
- Valuation confidence degrades when assumptions are missing.
- Memo Markdown and HTML include Thesis Snapshot.
- Memo valuation section includes valuation confidence.
- Research pack includes Research Quality Overview and coverage columns.
- Offline demo smoke test still passes.
- Release doctor passes for `0.12.0`.

## Release Criteria

v0.12.0 is ready when:

1. A user can track thesis, risks, triggers, and questions from CSV into memo and pack outputs.
2. Valuation scenarios show assumption completeness and margin-of-safety context without recommendation language.
3. Documentation explains the research method clearly enough for a GitHub visitor to understand how to use the tool.
4. Local tests and GitHub Actions pass for the release tag.
