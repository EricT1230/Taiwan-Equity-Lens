# v0.38.0 Gate Blocker Workflow Design

## Goal

v0.38.0 makes the Handoff Quality Gate actionable from the first dashboard screen. A user should not need to understand the detailed review-action table before handling the Top 3 blockers.

## Scope

- Keep `review_action_state.json` as the only state store.
- Keep the detailed review-action table as the source of truth for rows and API payloads.
- Render Expert Console blockers as task cards with:
  - expert lens,
  - severity,
  - priority,
  - stock context,
  - problem copy,
  - suggested handling copy,
  - direct action buttons.
- Add console-level Top 3 batch controls.
- In served dashboard mode, console buttons call the existing review-action API.
- In static dashboard mode, console buttons copy matching CLI commands.

## Non-Goals

- Do not add a second task ledger.
- Do not add investment recommendations or buy/sell/hold language.
- Do not change review-action status values.
- Do not remove the detailed review-action table.

## Acceptance Tests

- Dashboard renders Top 3 task cards with `問題` and `建議處理` copy.
- Dashboard renders console action buttons for `done` and `deferred`.
- Dashboard renders Top 3 batch buttons.
- Static console action buttons include copyable `research action set` commands.
- Served console action buttons reuse `updateReviewActionState(...)`.
- After a served update, the review-action summary and Expert Console resync from the same row data.
- Existing dashboard, dashboard-server, and review-action tests continue to pass.
