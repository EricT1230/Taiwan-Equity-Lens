# v0.47.0 Evidence Composer Design

v0.47.0 moves the evidence step into the dashboard. v0.46.0 made the Expert Agent Console show one primary next button; this release makes that step capable of creating the required evidence file and updating action state without leaving the served dashboard.

## Expert Review Lenses

- Product / UX: the operator should not need to understand prompt dialogs or manually create a markdown file before clearing the primary blocker.
- Finance research workflow: every generated artifact remains evidence-control and handoff-quality language only, not investment advice.
- Engineering: the endpoint must write only inside served dashboard directories, update the existing review-action state store, and return refreshed handoff gate counts.

## Scope

- Add an Evidence Composer inside the Next Action Workbench when the primary blocker requires handoff evidence and the dashboard is served with API mode enabled.
- Pre-fill:
  - note,
  - reviewer,
  - evidence file path,
  - evidence summary.
- Add `POST /api/evidence/compose-and-set` to:
  - validate the state and evidence paths against the served roots,
  - write a local evidence markdown stub,
  - update `review_action_state.json`,
  - return evidence file metadata plus refreshed gate status.
- Keep static dashboards on the existing copy-CLI-command path.
- Add served HTTP coverage that writes evidence and verifies a re-rendered ready gate.

## Non-Goals

- No new state store.
- No remote evidence upload.
- No live market-data, brokerage, or portfolio integration.
- No buy/sell/hold language, target prices, sector attractiveness ranking, or allocation guidance.

## Stable Hooks

- `data-evidence-composer="true"`
- `data-evidence-composer-note="true"`
- `data-evidence-composer-reviewer="true"`
- `data-evidence-composer-url="true"`
- `data-evidence-composer-summary="true"`
- `data-evidence-composer-submit="true"`
- `data-evidence-composer-result="true"`
