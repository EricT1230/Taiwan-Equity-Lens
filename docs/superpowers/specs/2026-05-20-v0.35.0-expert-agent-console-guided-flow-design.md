# v0.35.0 Expert Agent Console / Guided Flow Design

## Goal

When a user opens the dashboard, the first research-facing screen should immediately answer five questions:

- Can the current research be handed off, or is it blocked?
- What are the top 3 items to handle first?
- Which expert lens raised each blocker?
- What is the next button or action to press?
- Are the outputs investment advice?

The answer to the last question must always be no. Dashboard copy should state that outputs are research workflow aids only and do not constitute investment advice.

## Premortem Risks Addressed

- Death cause 1: silent quality or correctness degradation.
- Death cause 3: undefined win condition.
- Death cause 4: gap between research output and handoff.
- Death cause 5: over-trusting expert agents without an operating flow.

## Scope

v0.35.0 should not add another expert model or scoring layer first. It should turn the existing review-action queue, action state overlay, and fundamental expert review findings into a guided console.

## Dashboard Requirements

- Add a visible `Expert Agent Console` section before the detailed research and review-action tables.
- Show a handoff verdict:
  - blocked when open review actions remain.
  - ready for manual handoff when the full state-overlaid review-action queue has no open actions.
- Show the top 3 open actions across the review-action queue after applying saved review-action state.
- For every top action, show:
  - stock id and optional company name.
  - expert lens label.
  - severity.
  - action message.
  - next-step button that filters the existing review-action table to the matching action.
- Preserve stable machine-facing dashboard hooks. Existing `data-review-*` and `data-review-action-*` attributes must remain compatible.
- Add new stable hooks for the console:
  - `data-expert-agent-console="true"`
  - `data-expert-console-readiness="true"`
  - `data-expert-console-top-actions="true"`
  - `data-expert-console-focus-category`
  - `data-expert-console-focus-search`
  - `data-expert-console-non-advice="true"`

## Acceptance Tests

- `render_dashboard_html` includes the Expert Agent Console when a research summary has review actions.
- The console shows blocked handoff status with open actions.
- The console limits the guided list to the top 3 open actions.
- The console maps `fundamental_review` to the fundamental expert label.
- The console includes a non-investment-advice notice.
- Console buttons wire to JavaScript that focuses the existing review-action queue instead of creating a separate state system.
- Existing review-action tests continue to pass.
