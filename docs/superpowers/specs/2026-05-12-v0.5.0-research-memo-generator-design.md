# v0.5.0 Research Memo Generator Design

## Goal

v0.5.0 adds a research memo generator that turns existing analysis JSON and research workflow context into readable local Markdown and HTML notes.

## Product Positioning

Taiwan Equity Lens should remain a local research workflow tool. The memo generator should summarize available facts, assumptions, warnings, and follow-up questions. It must not write investment advice, buy/sell/hold language, portfolio allocation, price targets as recommendations, or final decision labels.

The memo should help a user answer:

1. What does the latest generated analysis say?
2. Which metrics and valuation assumptions are worth reviewing?
3. What data quality or reliability warnings need attention?
4. What manual checks should be done before the research state can move forward?
5. Where are the source JSON, report HTML, workflow summary, and research summary files?

## Scope

Included:

- Markdown memo output for a single stock from an existing `*_raw_data.json`
- Optional HTML memo output using the same content model
- Batch memo generation from `research_summary.json`
- Research-workflow integration that writes memos under `research-dist/memos/`
- Dashboard discovery and links for generated memos
- Documentation, example commands, release notes, and tests

Excluded:

- LLM-generated prose
- investment recommendation labels or decision scoring
- external news, technical analysis, or chip-flow sources
- PDF or DOCX export
- hosted web app or database
- changing existing financial metric formulas

## User Experience

### Single Memo

Generate a memo from one analysis JSON:

```powershell
python -m taiwan_stock_analysis.cli memo dist/2330_raw_data.json --output memos/2330_memo.md
```

Optional HTML output:

```powershell
python -m taiwan_stock_analysis.cli memo dist/2330_raw_data.json --output memos/2330_memo.html --format html
```

### Research Memo Batch

Generate memos from a research workflow directory:

```powershell
python -m taiwan_stock_analysis.cli research memo research.csv --workflow-dir research-dist --output-dir research-dist/memos
```

The command should read `research-dist/research_summary.json`, find generated stock JSON files under workflow report directories, and write one memo per stock when JSON exists.

### Research Run Integration

The existing command:

```powershell
python -m taiwan_stock_analysis.cli research run research.csv --output-dir research-dist --offline-prices
```

should generate research summaries and memo files as part of the run unless the user passes `--skip-memos`.

## Memo Content

Each memo should contain deterministic sections:

1. Title
   - stock id and optional company name
2. Research Metadata
   - category, priority, research state, notes when available
3. Data Reliability
   - workflow status, reliability status, data warnings, retry hints
4. Latest Metrics Snapshot
   - latest year, revenue, gross margin, net margin, ROE, EPS, debt-to-equity, free-cash-flow margin when available
5. Valuation Context
   - PE, PB, dividend yield, EPS assumptions, target-price scenarios as scenario outputs only
6. Quality Scorecard
   - total score, confidence, dimension labels and scores
7. Diagnostics
   - data issues and missing-field warnings
8. Review Checklist
   - deterministic checklist derived from warnings and missing assumptions
9. Source Links
   - analysis JSON path, report HTML path, workflow summary path, research summary path when available
10. Disclaimer
   - explicit statement that the memo is research support and not investment advice

## Data Model

Use plain dictionaries so the memo renderer can be fed by CLI paths or tests without extra dependencies.

Recommended context shape:

```json
{
  "analysis_path": "research-dist/reports/2330_raw_data.json",
  "report_path": "research-dist/reports/2330_analysis.html",
  "workflow_summary_path": "research-dist/workflow_summary.json",
  "research_summary_path": "research-dist/research_summary.json",
  "research_item": {
    "stock_id": "2330",
    "company_name": "TSMC",
    "category": "Semiconductor",
    "priority": "high",
    "research_state": "review",
    "notes": "Add valuation assumptions before final review",
    "workflow_status": "ok",
    "reliability_status": "warning",
    "attention_reasons": ["data reliability is warning"]
  }
}
```

## Architecture

Add a focused memo module:

- `src/taiwan_stock_analysis/memo.py`
  - loads analysis JSON
  - builds memo context from optional research summary
  - renders Markdown
  - renders HTML from the same structured sections
  - writes single memo and batch memos

Wire it through:

- `src/taiwan_stock_analysis/cli.py`
  - adds top-level `memo`
  - adds `research memo`
  - adds `--skip-memos` to `research run`
- `src/taiwan_stock_analysis/dashboard.py`
  - discovers `*_memo.md` and `*_memo.html`
  - renders memo links in the dashboard

No new third-party dependencies should be introduced.

## Output Rules

- Markdown should be stable and readable in GitHub.
- HTML should be static and use restrained styling consistent with existing outputs.
- Missing values should render as `-`.
- Target prices must be labeled as scenarios, not recommendations.
- Review checklist items should be deterministic and based only on available data:
  - review data reliability warning
  - verify missing valuation assumptions
  - inspect diagnostics warnings
  - review source filings manually
  - confirm research state and notes

## Error Handling

- Missing analysis JSON should raise a clear `FileNotFoundError`.
- Invalid JSON should raise a clear `ValueError`.
- Batch memo generation should skip missing per-stock JSON and record skipped stock ids in a summary.
- A single missing optional section should not prevent memo rendering.
- Research memo generation should not fail the entire research run if one memo cannot be produced; the failure should be recorded in `memo_summary.json`.

## Memo Summary

Batch generation should write:

```json
{
  "output_dir": "research-dist/memos",
  "generated": [
    {
      "stock_id": "2330",
      "markdown_path": "research-dist/memos/2330_memo.md",
      "html_path": "research-dist/memos/2330_memo.html"
    }
  ],
  "skipped": [
    {
      "stock_id": "9999",
      "reason": "analysis JSON not found"
    }
  ]
}
```

## Testing

Required coverage:

- render Markdown memo with research metadata, metrics, valuation context, reliability, checklist, and disclaimer
- render HTML memo with escaped user-provided values
- missing optional sections render as `-` or empty-state text
- invalid JSON raises `ValueError`
- single CLI `memo` writes Markdown and HTML
- `research memo` writes memo files and `memo_summary.json`
- `research run` writes memos by default
- `research run --skip-memos` skips memo generation
- dashboard discovers and links memo outputs
- existing tests continue passing

## Release Criteria

v0.5.0 is ready when:

1. Users can generate a single Markdown or HTML memo from analysis JSON.
2. Users can generate memos from a research workflow directory.
3. `research run` produces memos unless disabled.
4. Dashboard links to memo outputs.
5. Memo text avoids recommendation language and includes a disclaimer.
6. Tests cover memo rendering, CLI routes, research integration, dashboard discovery, and error handling.
7. Documentation explains the memo workflow clearly.
