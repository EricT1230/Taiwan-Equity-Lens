# v0.21.0 Review Action State Restore Design

## Goal

Add a safe CLI restore path for review-action state backups created by v0.20.0. A user should be able to recover a previous `review_action_state.json` from a `.bak-*` file without manually copying files.

## Command

Add a new subcommand:

```bash
python -m taiwan_stock_analysis.cli research action restore STATE_PATH BACKUP_PATH
```

- `STATE_PATH` is the target `review_action_state.json` to restore.
- `BACKUP_PATH` is the backup file to restore from.
- The user must explicitly provide the backup path. v0.21.0 will not auto-select the latest backup.

## Restore Behavior

1. Verify `BACKUP_PATH` exists.
2. Verify `BACKUP_PATH` is valid review-action state JSON using the existing state loader.
3. If `STATE_PATH` exists, verify the current state file is valid review-action state JSON.
4. If `STATE_PATH` exists and is valid, create a timestamped backup of the current file before restore.
5. Copy the original bytes from `BACKUP_PATH` to `STATE_PATH`.
6. Print the current-state backup path when one is created.
7. Print the restored target path after success.

Expected success output:

```text
Backup review action state: research-dist/review_action_state.json.bak-20260516T180000Z
Restored review action state: research-dist/review_action_state.json
```

If `STATE_PATH` does not exist, restore still succeeds after validating `BACKUP_PATH`, and no current-state backup line is printed.

## Error Handling

- Missing `BACKUP_PATH`: return `1`, print `Warning: backup review action state does not exist: PATH`, and do not write.
- Invalid `BACKUP_PATH`: return `1`, print the existing loader warning with backup context, and do not write.
- Invalid existing `STATE_PATH`: return `1`, print the existing loader warning with current-state context, and do not write.
- Backup creation failure before restore should abort the restore and propagate the exception through the CLI error path.
- Restore should preserve backup bytes exactly rather than reserializing JSON.

The restore command intentionally rejects an invalid current state file instead of overwriting it. That keeps restore symmetric with v0.20.0 write commands, which avoid writing over invalid state JSON.

## Code Shape

Add a helper in `review_action_state.py`:

```python
restore_review_action_state(path: Path, backup_path: Path) -> tuple[Path, Path | None]
```

The helper should:

- validate the backup file with `load_review_action_state`,
- validate the current file when it exists,
- call `backup_review_action_state(path)` before overwrite when `path` exists,
- copy `backup_path.read_bytes()` to `path.write_bytes(...)`,
- return `(path, current_backup_path)`.

The CLI should add `research action restore STATE_PATH BACKUP_PATH`, catch `ValueError`, print `Warning: ...`, and return `1`.

## Tests

Add unit coverage for:

- restoring preserves backup bytes exactly,
- restoring missing target creates no current-state backup,
- restoring existing target creates a timestamped backup of the pre-restore bytes,
- missing backup raises a clear `ValueError`,
- invalid backup raises a clear `ValueError`,
- invalid current state raises a clear `ValueError` and does not overwrite it.

Add CLI coverage for:

- successful restore prints backup and restore lines,
- missing target restore prints only the restore line,
- invalid backup returns `1` and leaves the target unchanged,
- invalid current state returns `1` and leaves both files unchanged.

## Non-Goals

- No dashboard restore controls.
- No automatic latest-backup selection.
- No backup cleanup or retention policy.
- No merge or partial restore of individual review-action entries.

## Acceptance Criteria

- `research action restore` restores from an explicit backup path.
- Existing valid state files are backed up before restore.
- Missing target state files can be restored from a valid backup without creating an extra backup.
- Invalid backup files never overwrite the target.
- Invalid current state files are not overwritten.
- Restored files preserve the selected backup's original bytes exactly.
- Focused state/CLI tests, full suite, Python 3.10 smoke, release doctor, and a restore demo all pass.
