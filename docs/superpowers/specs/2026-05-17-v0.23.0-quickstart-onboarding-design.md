# v0.23.0 Quickstart Onboarding Design

## Goal

Make the first successful run easier to discover. Taiwan Equity Lens already has a complete offline demo workflow, but a new user currently has to read through several sections before knowing which command to run and which outputs to inspect first.

## Scope

v0.23.0 is a docs-only onboarding release.

Add short, copyable quickstart guidance for:

1. Running the full offline demo.
2. Opening the dashboard.
3. Inspecting review actions.
4. Trying state persistence and backup listing.

## Recommended First Run

The primary quickstart command remains the existing offline workflow:

```powershell
python -m taiwan_stock_analysis.cli research run examples/research.csv --fixture-root examples/fixtures --output-dir demo-dist --offline-prices
```

This command is preferred because it:

- avoids live network dependencies,
- produces every major output surface,
- demonstrates research summaries, reports, memos, packs, comparison, dashboard, and review actions,
- finishes quickly with synthetic fixture data.

## Follow-Up Commands

After the demo run, document these next checks:

```powershell
python -m taiwan_stock_analysis.cli research action list demo-dist/research_summary.json --state demo-dist/review_action_state.json
python -m taiwan_stock_analysis.cli research action report demo-dist/research_summary.json --state demo-dist/review_action_state.json
python -m taiwan_stock_analysis.cli research action set demo-dist/review_action_state.json 2330 source-audit-manual-review --status done --note "checked source freshness"
python -m taiwan_stock_analysis.cli research action backups demo-dist/review_action_state.json
```

The docs should explain that the first `set` creates the state file and a later state-changing command will create a backup.
