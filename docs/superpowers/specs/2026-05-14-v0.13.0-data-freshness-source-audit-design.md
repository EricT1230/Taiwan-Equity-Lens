# v0.13.0 Data Freshness & Source Audit Design

## Goal

v0.13.0 adds deterministic source-audit and data-freshness context so every research workflow can explain where its data came from, when it was produced, whether it is stale, and what needs follow-up.

## Product Positioning

Taiwan Equity Lens already produces usable research artifacts. The next quality step is not more indicators; it is stronger auditability. A user should be able to open a report, memo, pack, or workflow summary and answer:

- Was this generated from live fetches or fixtures?
- Were prices fetched live or intentionally offline?
- When was this artifact generated?
- Is the data old enough to require a refresh?
- Which stocks require source review before handoff?

The feature remains deterministic and local. It does not automatically decide whether to invest, and it does not add external news, analyst estimates, or recommendation labels.

## Scope

Included:

- A new focused `freshness.py` module for source mode, generated-at parsing, age calculation, and stale-status classification.
- Analysis metadata fields that distinguish `live`, `fixture`, and `unknown` financial-statement sources.
- Workflow-level `source_audit` summary derived from generated reports, price rows, and run metadata.
- Research summary and pack output that surface source-audit status and stale items.
- Dashboard visibility for source-audit warnings.
- CLI/docs/release updates for `v0.13.0`.

Excluded:

- New external data vendors.
- Scheduler or background refresh automation.
- Automatic refetch retries.
- Timezone-heavy market-calendar logic.
- Any buy/sell/hold/allocation recommendation wording.

## Freshness Model

Add a small payload shape:

```json
{
  "generated_at": "2026-05-14T12:00:00Z",
  "source_mode": "fixture",
  "age_days": 0,
  "stale_after_days": 30,
  "status": "fresh",
  "review_reason": ""
}
```

Statuses:

- `fresh`: generated timestamp is valid and age is within threshold.
- `stale`: age is greater than threshold.
- `unknown`: timestamp is missing or invalid.
- `manual_review`: source mode or source payload indicates user review is required, such as offline price data.

Default thresholds:

- Financial statements: 120 days.
- Market prices: 7 days.
- Workflow artifacts: 30 days.

These are deterministic review thresholds, not financial rules.

## Source Mode Design

Single-stock analysis metadata adds:

```json
{
  "source_mode": "fixture",
  "source_review": {
    "status": "manual_review",
    "reason": "financial statements loaded from fixture files"
  }
}
```

Rules:

- `fixture_dir` present means `source_mode = "fixture"`.
- No fixture means `source_mode = "live"`.
- Missing or invalid metadata means `source_mode = "unknown"` downstream.
- Fixture mode is not an error; it is auditable and usually expected for offline demos.

Price audit derives from valuation CSV rows:

- `price_status = ok` -> price source is fresh if date is within threshold.
- `price_status = warning` with offline mode -> `manual_review`.
- Missing price date -> `unknown` or `manual_review` depending on status.

## Workflow Source Audit

`workflow_summary.json` adds:

```json
{
  "source_audit": {
    "status": "manual_review",
    "counts": {
      "fresh": 2,
      "stale": 0,
      "unknown": 0,
      "manual_review": 2
    },
    "items": [
      {
        "stock_id": "2330",
        "financial_statement": {
          "source_mode": "fixture",
          "status": "manual_review",
          "review_reason": "financial statements loaded from fixture files"
        },
        "price": {
          "source_mode": "offline",
          "status": "manual_review",
          "review_reason": "offline price mode"
        }
      }
    ]
  }
}
```

Overall status priority:

1. `stale`
2. `unknown`
3. `manual_review`
4. `fresh`

If any stale item exists, overall status is `stale`. If no stale but unknown exists, status is `unknown`. If only manual-review items exist, status is `manual_review`. Otherwise `fresh`.

## Research Summary and Pack Design

Research summary items inherit source-audit status for their stock:

```json
{
  "stock_id": "2330",
  "source_audit_status": "manual_review",
  "source_audit_reasons": [
    "financial statements loaded from fixture files",
    "offline price mode"
  ]
}
```

Research packs add a `Source Audit Overview` section with:

- overall source-audit status
- counts by freshness status
- stale or unknown items
- manual-review items

This keeps the pack useful as the handoff artifact for what needs checking next.

## Dashboard Design

Dashboard summary cards show source audit when a workflow summary includes it:

- overall status
- counts
- top review items

No interactive UI is required in v0.13.0.

## Error Handling

- Missing source-audit fields in older workflow summaries must render as absent, not broken.
- Invalid timestamps classify as `unknown`.
- Missing JSON files in workflow audit classify as `unknown` for that stock.
- Fixture mode and offline price mode classify as `manual_review`, not failure.

## Testing

Required coverage:

- `freshness.py` classifies fresh, stale, unknown, and manual-review cases.
- `analyze()` metadata records `source_mode` for fixture and live paths.
- `run_watchlist_workflow()` writes `source_audit` in `workflow_summary.json`.
- Offline demo workflow produces manual-review source audit for fixtures/offline prices.
- Research summary inherits per-stock source audit status and reasons.
- Pack renders Source Audit Overview.
- Dashboard renders source-audit status when present and tolerates missing audit data.
- Release doctor passes for `0.13.0`.

## Release Criteria

v0.13.0 is ready when:

1. Every workflow summary can explain source mode and freshness status.
2. Offline demos clearly mark fixture/offline data as manual-review instead of pretending it is live.
3. Research packs and dashboards surface stale or unknown data before handoff.
4. Existing v0.12.0 workflows remain backward-compatible.
5. Local tests and GitHub Actions pass for `main` and the `v0.13.0` tag.
