# v0.39.0 Evidence-Required Handoff Design

v0.39.0 closes the gap found by the expert validation pass after v0.38.0: the dashboard made blockers easy to mark, but a marked blocker did not yet prove what was checked or who accepted the risk.

## Goal

When a user marks a high-risk blocker as handled, the system must require handoff evidence before the research can be treated as ready.

## Evidence Contract

Evidence-required action IDs include source-audit, workflow error/warning, reliability error/warning, unavailable valuation, fundamental-review blockers, blocked research state, and missing thesis/follow-up quality blockers.

For those actions, any non-open state must include:

- `note`: what was checked or why the blocker was accepted.
- `reviewer`: who made or accepted the decision.
- `evidence_url`: a file path or URL to the evidence.
- `updated_at`: write timestamp from the state layer.

If any field is missing, the Handoff Quality Gate returns `missing_evidence` blockers and `doctor handoff` fails.

## Interfaces

- CLI: `research action set` accepts `--reviewer` and `--evidence-url`.
- CLI: `research action list` shows note, reviewer, evidence URL, and updated timestamp.
- Dashboard API: `/api/review-actions/set` accepts and returns evidence fields.
- Served dashboard: evidence-required non-open updates prompt for note, reviewer, and evidence URL before writing state.
- Static dashboard: command details show the extra CLI flags needed for evidence-required blockers.

## Non-Goals

- This release does not upload or validate evidence files.
- This release does not change the investment-advice disclaimer.
- This release does not add portfolio recommendations, buy/sell/hold labels, or ranking-based investment output.
