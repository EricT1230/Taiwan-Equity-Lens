# v0.20.0 Release Notes

## Summary

v0.20.0 adds automatic backups before existing review-action state files are rewritten by CLI write commands. This gives users a manual recovery point before changing persisted review-action decisions.

## Added

- Timestamped backups for `research action set` when updating an existing valid state file.
- Timestamped backups for `research action prune-stale --write` when pruning an existing valid state file.
- Backup path output in CLI write commands.
- Backup filename collision handling for multiple writes in the same second.

## Changed

- `research action set` now refuses to overwrite invalid state JSON. It prints a warning and exits non-zero instead.

## Verification

```powershell
python -m unittest tests.test_review_action_state tests.test_cli -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.20.0
```

## Notes

This release does not add restore or backup cleanup commands. Backups are plain files next to `review_action_state.json` and can be inspected or restored manually.
