# v0.10.0 - Release Quality Gate

v0.10.0 adds a local release readiness check for version metadata, release notes, documentation links, and CI visibility.

## Highlights

- `doctor release` validates the release state before tagging.
- Version metadata is checked across `pyproject.toml`, README, CHANGELOG, and release notes.
- Local Markdown links in key public docs are checked for missing files.
- GitHub Actions no longer cancels the rest of the Python matrix after one version fails.

## Verify

```powershell
python -m taiwan_stock_analysis.cli doctor release --version 0.10.0
python -m unittest discover -s tests -v
$env:UV_NATIVE_TLS='true'
uv run --python 3.10 python -m unittest discover -s tests -v
```
