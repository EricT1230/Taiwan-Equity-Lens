# v0.27.0 Release Notes

## Summary

v0.27.0 adds `doctor demo --open`, a first-run shortcut that opens the generated dashboard only after the demo readiness check passes.

## Added

- `python -m taiwan_stock_analysis.cli doctor demo --output-dir demo-dist --open`.
- Dashboard opener support using the platform default browser mechanism.
- JSON output fields:
  - `opened_dashboard`
  - `open_error`
- CLI tests for successful open, failed readiness without open, JSON plus open, and opener failure.

## Changed

- `doctor demo --open` does not open anything when readiness fails.
- If readiness passes but dashboard opening fails, the command returns non-zero and prints a warning in text mode.
- `doctor demo --json --open` keeps stdout valid JSON and reports open status in JSON fields.
- Package version is now `0.27.0`.

## Verification

```powershell
python -m taiwan_stock_analysis.cli demo quickstart --output-dir .tmp-cli-test/v0.27-demo-open
python -m taiwan_stock_analysis.cli doctor demo --output-dir .tmp-cli-test/v0.27-demo-open --json --open
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.27.0
```

## Notes

The opener runs only after demo readiness passes. Tests patch the opener and do not launch a browser.
