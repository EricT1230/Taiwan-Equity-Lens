# v0.39.0 Release Notes

v0.39.0 makes handoff evidence explicit. A high-risk blocker can no longer disappear from the Handoff Quality Gate simply because it was marked `done`, `deferred`, or `ignored`; it must carry enough evidence for a reviewer to understand who checked it, what was checked, and where the proof lives.

## Added

- Evidence-required gate blockers for handled high-risk actions missing:
  - `note`
  - `reviewer`
  - `evidence_url`
  - `updated_at`
- `research action set --reviewer ... --evidence-url ...`.
- Dashboard API prompts for `note`, `reviewer`, and `evidence URL` before updating evidence-required blockers.
- Evidence-required badges and evidence metadata in Expert Agent Console tasks and Review Actions rows.
- Static dashboard hints showing the extra CLI flags needed for evidence-required blockers.

## Changed

- `doctor handoff` now includes `evidence-required gaps` in the Gate summary.
- `research action list` includes `note`, `reviewer`, `evidence_url`, and `updated_at` columns.
- Package version is now `0.39.0`.
- README and usage workflow release doctor examples now target `0.39.0`.

## Quality

- Added handoff gate coverage for handled-but-missing-evidence blockers.
- Added state overlay and CLI coverage for reviewer and evidence URL fields.
- Added served dashboard API and UI hook coverage for evidence prompts and persisted evidence output.
