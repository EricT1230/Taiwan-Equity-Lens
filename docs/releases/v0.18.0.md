# v0.18.0 Release Notes

## Summary

v0.18.0 adds a read-only audit layer for persisted review-action state. The new CLI report and dashboard health badges make it easier to see whether the sidecar state file still matches the current generated queue.

## Added

- `research action report` for deterministic state-health output.
- Status counts for current generated review actions after applying persisted state.
- Stale sidecar entry detection for state rows that no longer map to generated actions.
- Latest `updated_at` reporting across valid state entries.
- Next-open action listing in current queue order.
- Dashboard badges for state health, stale state count, and latest update time.

## Verification

```powershell
python -m unittest tests.test_review_action_state tests.test_cli tests.test_dashboard -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.18.0
```

## Notes

This release does not prune or rewrite stale state. It reports stale entries so the user can decide whether to keep, edit, or remove them manually.
