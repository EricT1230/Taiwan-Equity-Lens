# v0.45.0 Release Notes

v0.45.0 strengthens the served dashboard quality gate by testing the actual local HTTP path. The dashboard server is started on an ephemeral port, the rendered dashboard is fetched, a sector evidence-board button payload is posted to `/api/review-actions/set`, and the dashboard is fetched again to confirm the stock evidence row becomes deliverable.

## Added

- `create_dashboard_server()` for starting the dashboard HTTP server in tests and tooling.
- End-to-end served HTTP smoke coverage for:
  - GET `/`,
  - reading the rendered `補證並標記完成` button payload,
  - POST `/api/review-actions/set`,
  - `review_action_state.json` write verification,
  - post-write dashboard re-render verification.

## Changed

- `serve_dashboard()` now uses the same server factory as the HTTP smoke test.
- README, usage workflow, changelog, release notes, and package version now target `v0.45.0`.

## Non-Advice Boundary

This release verifies dashboard workflow plumbing only. It does not add investment recommendations, sector attractiveness rankings, buy/sell/hold decisions, target prices, or allocation guidance.

## Verification

```powershell
python -m unittest tests.test_dashboard_server -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.45.0
```
