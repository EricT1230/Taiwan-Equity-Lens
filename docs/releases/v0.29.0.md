# v0.29.0 Release Notes

## Summary

v0.29.0 adds an interactive localhost dashboard mode so review-action buttons can update state directly instead of only copying CLI commands.

## Added

- `python -m taiwan_stock_analysis.cli dashboard --scan-dir demo-dist --serve --port 8765 --open`
- Local dashboard server bound to `127.0.0.1` by default.
- `POST /api/review-actions/set` for updating review-action state.
- API-enabled dashboard buttons for `done`, `deferred`, `ignored`, and `reopen`.
- Page-level result text after a status update, including backup path when one is created.

## Changed

- Static dashboard files still copy the equivalent CLI command.
- Served dashboards call the local API and update the row status after a successful response.
- The API writes through the existing `set_review_action_state` implementation, preserving validation and backup behavior.
- Package version is now `0.29.0`.

## Safety

- The server is local-only by default.
- State paths are restricted to the served dashboard scan directories.
- Invalid statuses are rejected by the existing review-action state validation.

## Verification

```powershell
python -m unittest tests.test_dashboard tests.test_dashboard_server tests.test_cli.CliTests.test_main_dashboard_serve_starts_local_server -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.29.0
```
