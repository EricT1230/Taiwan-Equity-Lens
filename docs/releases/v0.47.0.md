# v0.47.0 Release Notes

v0.47.0 adds an Evidence Composer to the served dashboard. When the Next Action Workbench points at an evidence-required blocker, the dashboard can now pre-fill a local evidence stub, write the markdown file, update `review_action_state.json`, and show the refreshed handoff gate result in one flow.

## Added

- Evidence Composer inside the Next Action Workbench for served dashboards.
- New local API endpoint:
  - `POST /api/evidence/compose-and-set`
- Evidence markdown stub generation with:
  - stock/action title,
  - reviewer,
  - status,
  - review note,
  - evidence summary,
  - non-investment-advice notice.
- API response fields for the generated evidence path and whether the file was created or updated.
- Dashboard and served HTTP tests for the end-to-end evidence composer flow.

## Changed

- The first-screen guided workflow no longer has to rely on prompt dialogs or manual file creation for the primary evidence-required blocker.
- The composer reuses `review_action_state.json` as the only workflow state store.
- README, usage workflow, changelog, release notes, and package version now target `v0.47.0`.

## Non-Advice Boundary

This release creates review evidence for handoff workflow quality. It does not add investment recommendations, sector attractiveness rankings, buy/sell/hold decisions, target prices, or allocation guidance.
