# v0.21.0 Release Notes

## Summary

v0.21.0 adds a safe restore command for review-action state backups. Backups created by v0.20.0 can now be restored from the CLI without manually copying files.

## Added

- `research action restore STATE_PATH BACKUP_PATH` for restoring an explicit backup file.
- Validation for the selected backup before any write.
- Validation for the current state file before overwrite when it exists.
- Automatic backup of the current valid state before restore.
- Byte-preserving restore that copies the selected backup as-is.

## Changed

- Review-action backup files are now part of a full backup-and-restore CLI workflow.

## Verification

```powershell
python -m unittest tests.test_review_action_state tests.test_cli -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.21.0
```

## Notes

This release intentionally requires an explicit backup path. It does not add dashboard restore controls, automatic latest-backup selection, backup cleanup, or partial restore of individual review-action entries.
