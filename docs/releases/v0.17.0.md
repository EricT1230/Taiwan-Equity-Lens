# v0.17.0 Release Notes

## Summary

v0.17.0 makes review-action persistence easier to use from the static dashboard. Each Review Actions row now includes command buttons that copy the exact CLI command needed to mark an action done, deferred, ignored, or open again.

## Added

- Dashboard command buttons for `done`, `deferred`, `ignored`, and `reopen`.
- Clipboard copy behavior with a static HTML fallback command block.
- PowerShell-safe command argument quoting for state path, stock ID, and action ID.
- Dashboard display for persisted review-action `note` and `updated_at` metadata.

## Verification

```powershell
python -m unittest tests.test_dashboard -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.17.0
```

## Notes

Dashboard command buttons do not write files directly. They copy `research action set` commands, and the user applies state changes by running the copied command in PowerShell.
