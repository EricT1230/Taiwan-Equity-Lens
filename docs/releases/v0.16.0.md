# v0.16.0 Release Notes

## Summary

v0.16.0 adds persisted review-action completion state. Review actions remain deterministically generated in `research_summary.json`, while manual decisions live in a local `review_action_state.json` sidecar file that can survive reruns.

## Added

- `review_action_state.json` support for `open`, `done`, `deferred`, and `ignored` statuses.
- `research action set` CLI command for updating persisted action state.
- `research action list` CLI command for listing generated review actions with persisted state overlaid.
- Dashboard status column, status filter, state-count badge, and invalid-state warning.

## Verification

```powershell
python -m unittest tests.test_review_action_state -v
python -m unittest tests.test_cli tests.test_review_action_state -v
python -m unittest tests.test_dashboard tests.test_review_action_state -v
python -m taiwan_stock_analysis.cli doctor release --version 0.16.0
```

## Notes

`review_action_state.json` is workflow state. It may be committed for team handoff, but generated research summaries remain rebuildable without it.
