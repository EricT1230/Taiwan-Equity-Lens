# v0.48.0 Release Notes

v0.48.0 adds an Evidence Preview Quality Gate to the served dashboard. After the Evidence Composer writes a local evidence file and updates `review_action_state.json`, the dashboard now shows Reviewer Confidence, a check list, the next evidence-quality step, and a preview of the generated markdown.

## Added

- Deterministic evidence-quality assessment for dashboard-created evidence.
- New `/api/evidence/compose-and-set` response fields:
  - `evidence_quality`
  - `evidence_preview`
  - `reviewer_confidence_status`
  - `reviewer_confidence_ready`
- Dashboard result panel that shows:
  - Reviewer Confidence status,
  - quality checks,
  - next step,
  - generated evidence path,
  - markdown excerpt.
- Focused tests for high-confidence evidence and low-confidence default stubs.
- Served HTTP coverage that verifies the evidence composer response includes quality and preview metadata.

## Changed

- A structurally ready handoff can now still show `needs_review` if the evidence note, reviewer, or summary looks like a default stub.
- The Evidence Composer result now makes the generated evidence visible immediately instead of only reporting that the status was set.
- README, usage workflow, changelog, release notes, and package version now target `v0.48.0`.

## Non-Advice Boundary

This release reviews evidence quality for research handoff only. It does not add investment recommendations, sector attractiveness rankings, buy/sell/hold decisions, target prices, or allocation guidance.
