# v0.37.0 Release Notes

v0.37.0 adds a Handoff Quality Gate and records the project win condition. The goal is to make the dashboard and CLI answer the same operational question: can this research package move into human handoff review right now?

## Added

- Reusable `build_handoff_quality_gate(...)` helper for handoff readiness checks.
- `doctor handoff <research_summary.json>` CLI command.
- JSON output for handoff readiness automation with `doctor handoff --json`.
- Handoff blockers for:
  - open state-overlaid review actions,
  - stale `review_action_state.json` entries,
  - missing required gate actions inferred from source, workflow, reliability, valuation, fundamental-review, and research-state signals.
- Project win-condition document at `docs/project-win-condition.md`.

## Changed

- Expert Agent Console now uses the Handoff Quality Gate for readiness, Top 3 blockers, next-step copy, and blocker counts.
- Served dashboard console resync also blocks on stale state and missing gate-action counts.
- Package version is now `0.37.0`.
- README and usage workflow release doctor examples now target `0.37.0`.

## Quality

- Added unit coverage for handoff gate ready, open-blocker, stale-state, and missing-action paths.
- Added doctor coverage for text formatting, failures, and handled-state success.
- Added dashboard coverage for Handoff Gate data hooks and stale state surfaced in the Expert Agent Console.
- Review-action state loading now accepts UTF-8 BOM files produced by some Windows tooling.
- Added a v0.37.0 design note under `docs/superpowers/specs/`.
