# v0.25.0 Release Notes

## Summary

v0.25.0 adds `doctor demo`, a local readiness check for the bundled quickstart demo output. It verifies that the demo produced the expected handoff files and first review-action surfaces before a user starts reviewing the dashboard.

## Added

- `python -m taiwan_stock_analysis.cli doctor demo --output-dir demo-dist`.
- Required-file checks for `dashboard.html`, workflow and research summaries, memo and pack summaries, comparison output, and `valuation.csv`.
- JSON validation for `workflow_summary.json` and `research_summary.json`.
- Content checks for successful stock IDs, a populated review-action queue, and the dashboard `Review Actions` section.
- Repair output that points back to `demo quickstart` when the demo directory is incomplete.

## Changed

- README, usage workflow, and examples now recommend running `doctor demo` after `demo quickstart`.
- Package version is now `0.25.0`.

## Verification

```powershell
python -m taiwan_stock_analysis.cli demo quickstart --output-dir .tmp-cli-test/v0.25-demo-doctor
python -m taiwan_stock_analysis.cli doctor demo --output-dir .tmp-cli-test/v0.25-demo-doctor
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.25.0
```

## Notes

The demo doctor is read-only. It does not perform live data checks, open a browser, or repair generated files automatically.
