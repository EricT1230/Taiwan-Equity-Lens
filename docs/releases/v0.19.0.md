# v0.19.0 Release Notes

## Summary

v0.19.0 adds an explicit cleanup command for stale persisted review-action state. The command is dry-run by default, so users can inspect exactly what would be removed before writing changes to `review_action_state.json`.

## Added

- `research action prune-stale` for stale sidecar cleanup.
- Dry-run output that lists stale entries without modifying the state file.
- `--write` mode that rewrites the state file after removing only stale entries.
- Safe handling for missing state files, invalid state JSON, and malformed state entries.

## Verification

```powershell
python -m unittest tests.test_review_action_state tests.test_cli -v
python -m unittest discover -s tests -v
python -m taiwan_stock_analysis.cli doctor release --version 0.19.0
```

## Notes

Pruning is never automatic. `research run`, dashboard generation, and `research action report` remain read-only with respect to stale state.
