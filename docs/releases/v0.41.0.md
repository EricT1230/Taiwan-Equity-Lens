# v0.41.0 Release Notes

v0.41.0 moves the Handoff Evidence Pack from a CLI-only artifact into the dashboard workflow. The dashboard now tells the operator how to generate the pack, where the generated files are, and which evidence files need to be created before handoff.

## Added

- Served dashboard API endpoint for generating a Handoff Evidence Pack:
  - `POST /api/handoff-pack/write`
- Expert Agent Console workflow panel for Evidence Pack generation.
- Static dashboard fallback that copies the equivalent `research handoff-pack` command.
- Immediate output display for generated Markdown, HTML, and JSON summary paths.
- Missing and invalid evidence guidance with stock/action identifiers and suggested local evidence file paths.
- Industry rotation map preview that groups research items by category, shows research-delivery pressure from blockers and evidence gaps, and jumps back to the matching review-action tasks.

## Changed

- Discovered research summaries keep their source directory so dashboard gate rendering can validate local evidence references.
- The dashboard keeps the same review-action state file as the source of truth; Evidence Pack generation does not introduce a new state store.
- README, usage workflow, changelog, release notes, and package version now target `v0.41.0`.

## Non-Advice Boundary

The dashboard Evidence Pack workflow is a research operations and quality-control workflow. It does not provide investment advice, buy/sell/hold recommendations, target prices, or portfolio allocation guidance.

The industry rotation map follows the same boundary. Its color and ordering represent handoff blockers, evidence gaps, stale state, and review workload; they do not represent industry strength, price momentum, expected return, or allocation preference.
